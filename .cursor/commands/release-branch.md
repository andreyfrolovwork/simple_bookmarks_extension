# Создание релизной ветки и тега

Автоматизированный релиз: новая ветка, версия, тег, push в origin.

## Перед выполнением

1. **Проверь рабочую директорию** — не должно быть некоммиченных критичных изменений
2. **Убедись** что ты на `main` (или нужной базовой ветке)
3. **Утверди план** ниже

## План действий (для апрува)

Перед выполнением покажи пользователю, что будет сделано:

1. **Проверить origin** — получить последний тег (`v*`) и последнюю ветку `release/v*`
2. **Определить новую версию** — следующий patch после последнего тега (например v2.0.49 → v2.0.50)
3. **Создать ветку** `release/vX.Y.Z` от текущего HEAD
4. **Обновить** `package.json` → версия `X.Y.Z`
5. **Закоммитить** изменение версии в release-ветку
6. **Создать тег** `vX.Y.Z` на этом коммите
7. **Push** в origin: ветка `release/vX.Y.Z` и тег `vX.Y.Z`

## Выполнение

После утверждения плана выполни пошагово:

```bash
# 1. Обновить данные с origin
git fetch origin

# 2. Определить новую версию (последний тег + patch)
# v2.0.49 → 2.0.50

# 3. Создать release-ветку
git checkout -b release/vX.Y.Z

# 4. Обновить package.json (версия)
# 5. Коммит
git add package.json && git commit -m "chore: bump version to X.Y.Z"

# 6. Тег
git tag vX.Y.Z

# 7. Push
git push origin release/vX.Y.Z
git push origin vX.Y.Z
```

## Важно

- Версионирование: SemVer patch (X.Y.Z → X.Y.(Z+1))
- Базовая ветка по умолчанию: `main`
- Если `package.json` уже на новой версии — не менять, использовать её для тега
